# OpenAI kills its dedicated coding model Codex again, folding it into GPT-5.5

**Source**: The Decoder | **Published**: 2026-04-26 | **Topics**: Large Language Models and Foundation Models

## Summary
OpenAI가 전용 코딩 모델 Codex를 또다시 폐기하고 그 기능을 GPT-5.5에 통합했다. GPT-5.5는 더 강력한 에이전틱 코딩 능력과 낮은 토큰 사용량을 제공한다고 OpenAI는 밝혔다. 개발자들은 별도 모델 대신 단일 플래그십 모델을 사용하게 된다.

## Key Points
- OpenAI가 Codex 전용 모델을 GPT-5.5에 완전 통합
- 에이전틱 코딩과 낮은 토큰 사용량 개선이 통합의 주요 이유
- 개발자들은 별도 API 엔드포인트 없이 GPT-5.5만 사용하면 됨
- "Codex" 브랜드는 Claude Code 등 AI 코딩 에이전트 제품명으로 여전히 사용
- OpenAI의 제품 라인 단순화 전략의 일환

## Why This Matters
전용 코딩 모델 시대에서 범용 에이전틱 모델 시대로의 전환을 보여주는 사례로, AI 코딩 도구 생태계 전반에 제품 통합 압력이 높아질 전망이다.
